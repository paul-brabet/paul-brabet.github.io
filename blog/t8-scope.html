Scope

13 April 2017

Explain to a non-tech friend what 'scope' is, and how it works in JavaScript.
Scope is the idea that certain parts of the code can be read by the computer, and others not unless specific circumstances apply. In javascript, variable visibility exists in either global scope or local scope form. The former is always visible, and the latter is only visible inside a function. In order to remember which is which, I like to think of global scope as looking at the overall picture. At this high level I can only see the global variables; it is only when I start zoooming in to view things at a local level that I start seeing local variables.
What does this look like in javascript? //Below is a global variable var globalVariable = 'visible'; //And I can call it at any point, like right now console.log(globalVariable); //Or I can call it inside a function var visibleInsideAFunction () { console.log(globalVariable) }; visibleInsideAFunction; //Below is a local variable. It is local to a function I have named 'private' var private = function () { var localVariable = 'sneaky sneaky'; console.log(localVariable); }; //If I call on this local variable without using the relevant function like below... console.log(localVariable); //...the result will be null. This is because it can't be read //But if I call the function that localVariable is local to, it will work private;
